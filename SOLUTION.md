# SMILES-2026 Hallucination Detection Solution

## Task

The task is binary hallucination detection for responses generated by
`Qwen/Qwen2.5-0.5B`. The official `solution.py` extracts hidden states for
`prompt + response`, trains a lightweight probe, evaluates labelled data, and
saves predictions for `data/test.csv`.

Labels:

- `0`: truthful response
- `1`: hallucinated response

## Modified Components

Only participant-editable files are changed:

- `aggregation.py`
- `probe.py`
- `splitting.py`

The fixed infrastructure files `solution.py`, `model.py`, and `evaluate.py`
are not modified.

## Final Approach

The final submitted model is intentionally compact:

- features: final transformer layer, last real token representation
- feature dimension: `896`
- probe: one-hidden-layer MLP
- hidden layer size: `256`
- scaler: `StandardScaler`
- loss: `BCEWithLogitsLoss`
- class imbalance handling: `pos_weight = n_neg / n_pos`
- fixed seed: `PROBE_SEED=19` by default
- validation split: original stratified train/validation/test split from the
  provided skeleton

This approach was selected because it achieved the best primary metric among
the tested variants while remaining simple and reproducible.

## Why This Works

The strongest hidden-state signal was the final layer's last token. This token
represents the model state after reading the full `prompt + response`, making it
a compact summary of the answer and its consistency with the context.

More complex representations were tested, but they tended to overfit on the
small labelled dataset (`689` samples). The final solution therefore prioritizes
generalization over feature volume.

## Validation Result

Best seed-search result:

| Feature Set | Probe | Seed | Feature Dim | Test-Split Accuracy | Test-Split F1 | Test-Split AUROC |
|---|---|---:|---:|---:|---:|---:|
| final layer, last token | MLP | 19 | 896 | 0.7500 | 0.8375 | 0.7371 |

The initial skeleton baseline was:

| Feature Set | Probe | Feature Dim | Test-Split Accuracy | Test-Split AUROC |
|---|---|---:|---:|---:|
| final layer, last token | MLP | 896 | 0.7404 | 0.7366 |

The final fixed-seed version improves the primary accuracy from `0.7404` to
`0.7500`.

## Experiments

The final model was chosen after controlled ablations:

| Stage | Summary | Test-Split Accuracy | Outcome |
|---:|---|---:|---|
| 0 | final layer, last token, original MLP | 0.7404 | strong baseline |
| 1 | multi-layer mean/last/max + LogisticRegression | 0.6865 | overfit/underperformed |
| 2 | trajectory + spectral covariance features | 0.6923 | did not improve |
| 3 | layer-wise mean+last probing | 0.7126 | below baseline |
| 4 | layer-wise last-token probing | 0.6981 | below baseline |
| 5 | tuned MLP with K-fold | 0.6866 | good AUROC, poor accuracy |
| 6 | original MLP with K-fold | 0.6952 | threshold/calibration issue |
| 7 | imbalance-aware MLP | 0.6894 | did not improve accuracy |
| 8 | data-driven tail-window features | 0.7170 | useful but below baseline |
| 9 | tail-window features, single split | 0.5962 | severe overfit |
| 10 | histogram gradient boosting | 0.7019 | near majority baseline |
| 11 | MLP seed ensemble | 0.7212 | stable but below baseline |
| 12 | layer-wise ensemble + trajectory/spectral scalars | 0.7212 | stable but below baseline |
| 13 | original MLP seed search | 0.7500 | final choice |

## Data Analysis Findings

The dataset is imbalanced:

- truthful (`0`): `206`
- hallucinated (`1`): `483`
- majority-class accuracy: about `0.701`

EDA showed strong response-side correlations:

- hallucinated responses are longer on average;
- hallucinated responses have lower normalized lexical overlap with the context;
- numeric text/structure features alone reached about `0.721` CV AUROC.

These findings motivated tail-window hidden-state experiments, but the compact
last-token representation still generalized best.

## Reproducibility

Recommended environment: Google Colab or Kaggle Notebook with GPU.

Commands:

```bash
git clone https://github.com/mart1ny/summer-school-application-china.git
cd summer-school-application-china
pip install -r requirements.txt
python solution.py
```

The official script generates:

- `results.json`
- `predictions.csv`

The default seed is fixed in `probe.py`:

```python
DEFAULT_PROBE_SEED = 19
```

It can be overridden for additional experiments:

```bash
PROBE_SEED=7 python solution.py
```
