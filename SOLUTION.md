# SMILES-2026 Hallucination Detection Solution

## Task

The task is binary hallucination detection for responses generated by
`Qwen/Qwen2.5-0.5B`. The official `solution.py` extracts hidden states for
`prompt + response`, builds fixed-size feature vectors, trains a lightweight
probe, evaluates it on labelled data, and saves predictions for `data/test.csv`.

Labels:

- `0`: truthful response
- `1`: hallucinated response

## Modified Components

Only participant-editable files were changed:

- `aggregation.py`
- `probe.py`
- `splitting.py`

The fixed infrastructure files `solution.py`, `model.py`, and `evaluate.py`
were not modified.

## Features

The solution uses hidden-state features from several transformer layers:

- final layer: `-1`
- nearby late layers: `-2`, `-4`
- middle/late layers: `-8`, `-12`

For each selected layer, three token aggregation methods are concatenated:

- mean pooling over real tokens
- last real token representation
- max pooling over real tokens

With Qwen hidden size `896`, this produces:

`5 layers * 3 pooling methods * 896 = 13440` features per sample.

Optional geometric features are implemented in `extract_geometric_features`,
but the official `solution.py` keeps `USE_GEOMETRIC = False`, so the submitted
baseline uses only the hidden-state aggregation above.

## Probe

The probe is a deterministic linear classifier:

- `StandardScaler`
- `LogisticRegression`
- L2 regularization
- `solver="liblinear"`
- `C=0.25`
- `random_state=42`

This is intentionally simple and robust for the small labelled dataset and
high-dimensional hidden-state features. No extra dependencies were added.

The classification threshold is selected using only labels available inside the
current fit:

- during fold evaluation: tuned on the validation split
- during final prediction generation: tuned on the final training data used by
`solution.py`

No competition test labels are used.

## Validation Strategy

`splitting.py` uses reproducible stratified evaluation:

- outer `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`
- each outer train part is split into train/validation with stratification
- validation split uses `random_state=42 + fold_id`

This gives a more stable estimate than a single random split while preserving
class balance.

## Reproducibility

Recommended environment: Google Colab or Kaggle Notebook with GPU.

Commands:

```bash
git clone <your-repository-url>
cd <repository-name>
pip install -r requirements.txt
python solution.py
```

The official script should generate:

- `results.json`
- `predictions.csv`

Local note: full end-to-end execution was not completed on the laptop because
hidden-state extraction for `Qwen/Qwen2.5-0.5B` heavily loaded the machine. The
code-level checks passed locally: Python compilation, aggregation shape checks,
split non-overlap checks, and probe API checks.

## Experiments and Rationale

Included:

- multi-layer aggregation instead of only the final layer
- mean, last-token, and max pooling concatenation
- standardized linear logistic probe
- stratified K-fold validation

Not included in the current baseline:

- neural MLP probe: higher overfitting risk on 689 labelled samples
- heavy feature selection or PCA: adds tuning complexity and extra failure modes
- geometric features in the official run: `solution.py` keeps
  `USE_GEOMETRIC = False`, and fixed infrastructure was intentionally left
  unchanged
